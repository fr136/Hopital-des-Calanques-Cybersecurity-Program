# Campagne de Phishing Simulee

## Objectif

Evaluer et renforcer la capacite des agents a identifier les courriels frauduleux.

## Cadre

Les campagnes sont realisees a des fins pedagogiques, sans collecte de mots de passe reels et sans contenu offensif. Les resultats individuels ne sont pas utilises pour sanctionner.

## Frequence

Deux campagnes minimum par an, avec une campagne supplementaire possible apres incident ou hausse des signalements.

## Scenarios Fictifs

- fausse facture fournisseur pour les admissions ;
- fausse notification Microsoft 365 ;
- fausse demande RH ;
- faux document de livraison ;
- fausse alerte de quota de messagerie.

## Indicateurs

- taux d'ouverture ;
- taux de clic ;
- taux de saisie sur page fictive ;
- taux de signalement ;
- delai moyen de signalement ;
- progression par service.

## Actions Apres Campagne

- message de sensibilisation ;
- rappel des bons reflexes ;
- module court pour les services les plus exposes ;
- mise a jour des supports si un theme est mal compris.

## Reporting

Les resultats consolides sont presentes au comite cyber avec les actions prevues.
